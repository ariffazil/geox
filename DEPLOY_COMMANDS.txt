If the deploy script fails, execute these commands manually on the VPS:

───────────────────────────────────────────────────────────────────────────────
Step 1: SSH to VPS
───────────────────────────────────────────────────────────────────────────────
ssh srv1325122.hstgr.cloud

───────────────────────────────────────────────────────────────────────────────
Step 2: Navigate to GEOX directory
───────────────────────────────────────────────────────────────────────────────
cd /opt/arifos/geox

───────────────────────────────────────────────────────────────────────────────
Step 3: Backup current state
───────────────────────────────────────────────────────────────────────────────
docker commit geox_server pre-openai-adapter-$(date +%Y%m%d) 2>/dev/null || true
git stash push -m "pre-deployment-backup" 2>/dev/null || true

───────────────────────────────────────────────────────────────────────────────
Step 4: Pull latest code
───────────────────────────────────────────────────────────────────────────────
git fetch origin main
git reset --hard origin/main

───────────────────────────────────────────────────────────────────────────────
Step 5: Stop containers
───────────────────────────────────────────────────────────────────────────────
docker compose down --remove-orphans

───────────────────────────────────────────────────────────────────────────────
Step 6: Clean cache (CRITICAL for GUI rebuild)
───────────────────────────────────────────────────────────────────────────────
docker system prune -f --volumes
docker volume prune -f
docker builder prune -f

───────────────────────────────────────────────────────────────────────────────
Step 7: Rebuild with no cache
───────────────────────────────────────────────────────────────────────────────
export DOCKER_BUILDKIT=1
docker compose build --no-cache geox_server

───────────────────────────────────────────────────────────────────────────────
Step 8: Start containers
───────────────────────────────────────────────────────────────────────────────
docker compose up -d geox_server
sleep 5

───────────────────────────────────────────────────────────────────────────────
Step 9: Verify deployment
───────────────────────────────────────────────────────────────────────────────
curl https://geox.arif-fazil.com/health
curl https://geox.arif-fazil.com/health/details

───────────────────────────────────────────────────────────────────────────────
Step 10: Check logs if needed
───────────────────────────────────────────────────────────────────────────────
docker compose logs --tail=100 geox_server

═══════════════════════════════════════════════════════════════════════════════
Troubleshooting
═══════════════════════════════════════════════════════════════════════════════

If build fails:
  docker compose build --no-cache --progress=plain geox_server 2>&1 | tee build.log

If container won't start:
  docker compose logs geox_server | grep -i error

If GUI not updated:
  docker exec geox_server ls -la /app/geox-gui/dist/  # Check build output

Rollback:
  docker stop geox_server
  docker run -d --name geox_server_rollback [BACKUP_IMAGE]
