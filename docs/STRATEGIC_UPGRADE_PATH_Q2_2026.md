## 2. arifOS Core Upgrades (The Mind & Soul)

### A. Execution Kernel, Not Just Prompting
Move arifOS from an "advice layer" (tone control, symbolic guardrails) to a strict control plane with enforceable layers:
- Intent parser → Authority classifier → Risk classifier → Tool permission broker → Human approval gate → State ledger → Action validator → Post-action audit recorder.

### B. Formal State Machine
Every tool call must carry: `state origin`, `state transition reason`, `policy basis`, `actor identity`, `confidence`, and `rollback path`. Explicit states:
`INPUT_RECEIVED`, `INTENT_PARSED`, `RISK_CLASSIFIED`, `AUTHORITY_VERIFIED`, `TOOLS_SCOPED`, `PLAN_DRAFTED`, `HUMAN_APPROVAL_REQUIRED`, `EXECUTION_ALLOWED`, `EXECUTION_BLOCKED`, `RESULT_AUDITED`, `MEMORY_WRITE_ALLOWED`, `MEMORY_WRITE_BLOCKED`.

### C. Protocol-Level Separation of Suggestion vs. Action
Explicit action classes to prevent "assistant became operator" drift:
`informational`, `advisory`, `drafting`, `simulation`, `low-risk execution`, `high-risk execution`, `irreversible action`.

### D. Structured Uncertainty Protocol
Every nontrivial output must carry internal fields:
`confidence_score`, `uncertainty_type`, `data_missing`, `model_ambiguity`, `conflicting_evidence`, `out-of-domain`, `decision_required_by_human`, `safe_fallback`.

### E. Memory Governance
Create 4 strict memory classes:
- Ephemeral session memory
- Working memory
- Governed durable memory
- Restricted/private memory
*(Each write needs: source, confidence, authorizer, expiry, reversibility, sensitivity tag).*

## 3. GEOX MCP Core Upgrades (The Earth Witness Body)

### A. True Multimodal Ingestion
Ingest and reason over: Seismic sections, well logs, LAS/DLIS metadata, maps, structural surfaces, strat columns, reports, production trends.

### B. Physics and Geology Constraint Layer
Before final answers, pass through: Stratigraphic consistency, depth/time sanity, unit consistency, petrophysical plausibility, reservoir engineering plausibility, structural closure logic.

### C. Hypothesis Competition Engine
Subsurface decisions are rarely single-answer. Force GEOX to generate:
- Hypothesis A / B / C
- Evidence for each
- Uncertainty rank
- Decision sensitivity

### D. Spatial Reasoning Engine
Improve consistency for: Top/base relationships, fault juxtaposition, map-to-section consistency, horizon continuity, well-to-seismic tie reasoning, pressure compartment logic.

### E. Uncertainty-Aware Outputs
Include low/medium/high confidence maps, uncertainty intervals, and explicitly define the "next best data acquisition/analysis action" to reduce uncertainty.

## 4. Cross-Stack Enhancements

### A. Observability and Replay
Full trace system for every run logging input, intent, selected/blocked tools, evidence chain, policy state, etc. Plus: Replay mode, diff mode, failure clustering.

### B. Evaluation Harness (5 Benchmark Layers)
1. Unit tests
2. Workflow tests
3. Adversarial tests (prompt injection, false confidence attacks)

### C. Confidence Calibration & Human Escalation
Track confidence vs. actual correctness. Create an escalation matrix for missing data (pause/ask), conflicting evidence (present alternatives), high-risk operations (require human approval).
